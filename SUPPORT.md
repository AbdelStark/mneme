# Support

Mneme support happens in public GitHub issues unless the report contains private
or security-sensitive information.

## Use Issues For

- Reproducible bugs with commands, versions, inputs, and error output.
- Documentation corrections.
- Feature requests with a concrete use case and expected evidence.
- Questions about public APIs, CLI behavior, package metadata, or release
  artifacts.

## Use Security Reporting For

- Secrets, private datasets, private run outputs, or credentials exposed through
  logs, reports, artifacts, or examples.
- Vulnerabilities in store validation, redaction, receipt verification, remote
  response validation, or release artifacts.

Read [SECURITY.md](SECURITY.md) before sharing sensitive details.

## Not Support Channels

The project does not provide private deployment consulting, benchmark
certification, production authentication hosting, or confidentiality guarantees
for stores. Those requirements need operator-owned controls outside Mneme.
