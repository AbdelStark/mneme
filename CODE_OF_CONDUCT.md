# Code of Conduct

Mneme is a technical open-source project. Project spaces should stay useful for
engineering discussion: direct, specific, evidence-oriented, and respectful.

## Expected Behavior

- Focus critique on code, docs, claims, issues, and reproducible evidence.
- Keep disagreement precise and actionable.
- Respect privacy. Do not post private datasets, credentials, run logs, or
  personal information.
- Preserve the project's claim boundary. Do not pressure maintainers to present
  fixture-scale evidence as benchmark or production evidence.

## Unacceptable Behavior

- Harassment, threats, sustained personal attacks, or discriminatory language.
- Publishing private information without explicit permission.
- Spam, impersonation, or deliberately misleading benchmark, security, or
  capability claims.
- Repeatedly derailing issue or review threads after maintainers have asked for
  focus.

## Reporting

Report conduct concerns through the maintainer contact listed in
[SECURITY.md](SECURITY.md) or by opening a private GitHub security advisory when
the concern involves private data, credentials, or security-sensitive behavior.

Maintainers may remove comments, lock threads, block accounts, or decline
contributions when behavior makes the project less safe or less useful for
contributors.
